# 📊 SQL EDA Sales Analysis

## 📌 Project Overview

This project focuses on **Exploratory Data Analysis (EDA)** using SQL on a retail sales dataset. The objective is to analyze sales performance, customer behavior, and product insights by writing SQL queries that answer real-world business questions.

Although the original project was demonstrated using **Microsoft SQL Server**, this implementation was completed entirely in **MySQL Workbench**, with SQL Server-specific syntax adapted to MySQL.

---

## 🛠️ Tools & Technologies

## 📂 Dataset

The project uses three relational tables:

- **fact_sales** – Sales transactions
- **dim_customers** – Customer information
- **dim_products** – Product information

- Identified total sales, revenue, orders, customers, and products.
- Analyzed customer purchasing behavior.
- Explored product performance and sales distribution.
- Evaluated sales trends over time.

## 🚀 How to Run

1. Clone this repository.
2. Open MySQL Workbench.
3. Create the required database and tables.
4. Import the CSV files.
5. Execute the SQL script located in the `sql` folder.
6. Review the generated insights.
